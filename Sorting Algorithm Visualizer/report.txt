Key Technical Implementations
Generator Functions: Each sorting algorithm is implemented as an async generator that yields steps (compare, swap, render, sorted) for controlled visualization

DOM Manipulation: Tiles are dynamically created, updated, and reordered using insertBefore() and appendChild() for smooth position changes

Audio API: Uses Web Audio API for generating sine wave tones with adjustable frequency and duration

CSS Variables: Enables seamless dark/light theme switching across all components

Async/Await Pattern: Controls animation speed using setTimeout with configurable delays

🔧 Development Challenges & Solutions
Challenge 1: Real Tile Movement
Problem: Tiles needed to physically swap positions, not just change values
Solution: Used data-index attributes to track tile identity and insertBefore() for DOM reordering with CSS transitions

Challenge 2: Algorithm Pause/Resume
Problem: Users needed to stop sorting mid-execution
Solution: Implemented stopRequested flag checked in the generator loop, clearing timeouts properly

Challenge 3: Theme Switching
Problem: All components needed to update seamlessly
Solution: Used CSS custom properties (variables) with data-theme attribute on html element

Challenge 4: Audio Synchronization
Problem: Audio needed to play without blocking UI
Solution: Web Audio API with short duration tones (60–100ms) and low volume to avoid audio conflicts
